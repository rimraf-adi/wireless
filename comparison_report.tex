\begin{abstract}
This document presents a section-by-section comparison of four independently authored radio network planning reports for the same assignment.
All four reports design a 4G LTE network for a different town in Maharashtra using 3GPP TR~38.901 V18.0.0 and then simulate small-scale fading via the Clustered Delay Line (CDL) channel model.
The towns are:
\textbf{Kagal} (Kolhapur district) by Aditya Kinjawadekar,
\textbf{Ralegaon} (Yavatmal district) by Atharva S Chaudhari,
\textbf{Pombhurna} (Chandrapur district) by Prasad Patil (BT23ECE096), and
\textbf{Sakri} (Dhule district) by Tanmay Lakhambare (BT23ECE121).
Each section highlights \colorbox{matchcolor}{agreements}, \colorbox{diffcolor}{disagreements}, and \colorbox{uniquecolor}{unique features}
across the four reports.
\end{abstract}

\noindent\textbf{Critical Differences:}
\begin{itemize}
    \item \textcolor{accentred}{\textbf{Four different $L_{\max}$ computation approaches}} are now visible across the four reports:
    \begin{enumerate}
        \item \textbf{Kagal \& Ralegaon} ($L_{\max} = 158$~dB): Use antenna gain and cable loss in the extended budget.
        \item \textbf{Pombhurna} ($L_{\max} = 133$~dB): Uses thermal noise floor with NF, body loss, and penetration loss.
        \item \textbf{Sakri} ($L_{\max} = 140$~dB): Uses the simplest approach --- just $P_t - P_{r,\min} - L_{\text{sf}} = 150 - 10 = 140$~dB. Does \textbf{not} include antenna gain or cable loss in the link budget.
    \end{enumerate}
    \item Sakri uses a fading margin of \textbf{10~dB} (vs.\ 8~dB for the other three), citing it as a practical worst-case margin exceeding the 6~dB $\sigma_{\text{SF}}$ from 3GPP.
    \item The spread is now 25~dB between the highest (158~dB) and lowest (133~dB) $L_{\max}$ values.
\end{itemize}

\noindent\textbf{Observations:}
\begin{itemize}
    \item All four use the same UMa NLOS formula from 3GPP TR~38.901.
    \item \textcolor{accentred}{\textbf{Breakpoint distance discrepancy:}} Kagal and Ralegaon compute $d'_{BP} = 452.4$~m (using effective heights $h'_{BS} = 24$~m, $h'_{UT} = 0.5$~m as per 3GPP TR~38.901). \textbf{Sakri computes $d_{BP} = 900$~m} using the raw heights ($h_{BS} = 25$~m, $h_{UT} = 1.5$~m) without the 3GPP effective height correction. This is a \textbf{methodological error} --- 3GPP TR~38.901 defines $h'_{BS} = h_{BS} - h_E$ and $h'_{UT} = h_{UT} - h_E$ with $h_E = 1$~m for UMa, giving $d'_{BP} = 452.4$~m, not 900~m.

    \item \textbf{Sakri} uniquely takes $\text{PL} = \max(\text{PL}_{\text{LOS}}, \text{PL}_{\text{NLOS}})$ for final path loss evaluation, which is correct per the standard.
\end{itemize}

\subsection{Breakpoint Distance Error in Sakri Report}

\begin{center}
\fcolorbox{accentred}{uniquecolor}{%
\begin{minipage}{0.9\textwidth}
\textbf{Technical Error:} The Sakri report computes the breakpoint distance as $d_{BP} = \frac{4 h_{BS} h_{UT} f_c}{c} = \frac{4 \times 25 \times 1.5 \times 1.8 \times 10^9}{3 \times 10^8} = 900$~m. However, 3GPP TR~38.901 specifies that for UMa scenarios, \textbf{effective heights} must be used: $h'_{BS} = h_{BS} - h_E$ and $h'_{UT} = h_{UT} - h_E$ where $h_E = 1$~m. This gives $h'_{BS} = 24$~m and $h'_{UT} = 0.5$~m, yielding $d'_{BP} = 452.4$~m. The Sakri report uses the wrong heights, overestimating the breakpoint by a factor of $\approx$2. While this does not affect the NLOS path loss calculation (which does not depend on $d_{BP}$), it leads to incorrect LOS/NLOS regime selection near the breakpoint.
\end{minipage}
}
\end{center}

\subsection{Overall Assessment}

\begin{enumerate}
    \item \textbf{Link Budget Approach:} Four different approaches are now visible. Kagal/Ralegaon use an extended budget (158~dB). Sakri uses a simplified budget without antenna gain (140~dB). Pombhurna uses the most conservative approach with penetration/body losses (133~dB). All approaches are defensible, but the omission of antenna gain in Sakri's budget is questionable.

    \item \textbf{Network Architecture:} Kagal has the most elegant design (7-cell hex ring, no umbrella needed). Sakri demonstrates best practice in iterative design (redesigning umbrella cells when PL constraints fail). Ralegaon's 17-site design is over-engineered. Pombhurna is operationally tight.

    \item \textbf{Channel Simulation:} Kagal is the only report with a full 12-step CDL implementation producing simulation outputs. Sakri implements Steps~1--6 with code results but uses a single-antenna model. Ralegaon/Pombhurna implement Steps~1--6 with identical code.

    \item \textbf{Documentation Quality:} Sakri provides the best visual documentation (14 figures covering every design stage). Kagal provides the best technical depth (complete formulas, code, and simulation plots). Ralegaon provides the best geographic documentation (GPS coordinates).

    \item \textbf{Code Originality:} Kagal's code is independently authored. Sakri does not include code (cannot be verified). Ralegaon and Pombhurna share identical code.

    \item \textbf{Technical Errors:} Sakri has a breakpoint distance error. Ralegaon has an internal $L_{\max}$ inconsistency (report says 158~dB, code computes 133~dB).
\end{enumerate}
